    This Api is used to initiate eSign transaction with ESP.

    HTTP Request

    POST https://<<host>:<port>/<env>/api/2.1/initiateEsign

    Request HTTP Header Parameters

    Parameter Mn Type Description
    content-Type Y Anum MIME type of data found within a request or response message’s body.
    accept Y Anum Used by client to specify the acceptable media type.

    Request HTTP Body Parameters

    Parameter Mn Type Description
    client_code Y Anum Client Identifier. It will be the unique code assigned to hosted Client
    sub_client_code Y Anum Unique Identifier assigned to Sub Client like Sub AUA. Identifies the Sub AUA client who has initiated the request. For each Sub client platform will issue a client code. Should be same as client_code in case of client itself doing transaction.
    channel_code Y WEB_APP
    MOBILE_APP    		 Identifies the channel used by client to initiate request to the platform. Eg : MOBILE_APP, WEB_APP
    channel_version Y Anum Specifies the version of the Android SDK or Web SDK used by the client application. This is mandatory for Aadhaar Services. Eg : 1.0.1
    stan Y Anum System Trace Number. This will be a unique number generated by the client who is initiating the request and specifies in the request. This number is used to uniquelly identify a service request initiated from the client.
    client_ip N Captures the IP Address of the client device, if available.
    transmission_datetime Y Anum Transmission time in System Milli Seconds. Time at which request is initiated from client
    operation_mode Y SELF
    ASSISTED
    SYSTEM    		 Specifies the operation mode of the request.
    SELF - aadhaar holder doing the transaction.
    ASSISTED - where auth/ekyc is done in operator assisted mode.
    SYSTEM- where transaction is automated by system. eg. batch demographic authentications.
    run_mode Y REAL Specifies the run mode of the request. Eg : TRIAL, REAL,RETRY
    TRIAL - Indicates need for mock run of the transaction. No transaction posting will happen.
    REAL - Indicates the actual transaction posting request. Only REAL is allowed for doAadhaarService api
    actor_type Y Anum Specifies the type of actor. Based on the business use case of the client, value can be set.
    Eg: CUSTOMER, MERCHANT, RETAILER etc.
    user_handle_type Y Anum Indicates the type of the user identifier, who has initiated the request. Eg : MSISDN for mobile number, EMAIL for email address, USERNAME for username
    user_handle_value Y Anum Actual value of user_handle_type. email_address in case user_handle_type is EMAIL.
    location N Anum Specifies the geo location of the device that initiated the service request.
    function_code Y ESIGN Specifies the flavour of the api request.
    DEFAULT : Specifies the Default Behaviour.
    function_sub_code Y DEFAULT Specifies the flavour of the api request specified by the flavour specified by function_code.
    DEFAULT : Specifies the Default Behaviour.
    client_request_id Y Anum Created by client for tracking his request
    auth_mode Y Num "Authentication Mode being used for e-KYC. Authentication, either to be performed by ESP, or as already made by the ASP. Allowed values are: OTP = 1, Fingerprint = 2, IRIS = 3
    environment Y SANDBOX
    PREPROD
    PROD    		 Environment in which client is hitting the request
    success_url Y Anum This should be a valid url. We will redirect the response on this URL if it was a success. Example: https://mydomain.com/success.html
    failure_url Y Anum This should be a valid url. We will redirect response on this URL if it was a failure. This can be same as successUrl. Example: https://mydomain.com/failure.html
    consent Y Y
    N    		 The name which would be displayed in the digital signature. No name would be displayed in the signature if this is missing.
    esp_sdk_channel Y ANDROID
    WEB    		 Indicates whether ASP wants to perform transaction using ESP web page or ESP Andoid apk.

    Rsponse HTTP Header Parameters

    Parameter Mn Type Description
    content-Type Y Anum MIME type of response message’s body

    Rsponse HTTP Body Parameters

    Parameter Mn Type Description
    redirect_url C Anum ESP redirect URL. You have to redirect user to this url to complete the esign transaction. Mandatory in case of esp_sdk_channel is WEB
    esp_request_xml C Anum Base64 encoded request xml that should be passed to ESP android sdk. Mandatory in case of esp_sdk_channel is ANDROID
