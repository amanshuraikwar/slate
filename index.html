This Api is used to initiate eSign transaction with ESP.

HTTP Request

POST https://<<host>:<port>/<env>/api/2.1/initiateEsign

Request HTTP Header Parameters

Parameter (Y/N/C) Type Description
content-Type Y AlphaNumeric MIME type of data found within a request or response message’s body.
accept Y AlphaNumeric Used by client to specify the acceptable media type.
Content-Hash Y AlphaNumeric Salted Hash of http request body. hash=SHA-256(salt+JsonRequestBody)

Request HTTP Body Parameters

Parameter (Y/N/C) Type Description
client_code Y AlphaNumeric Client Identifier. It will be the unique code assigned to hosted Client
sub_client_code Y AlphaNumeric Unique Identifier assigned to Sub Client like Sub AUA. Identifies the Sub AUA client who has initiated the request. For each Sub client platform will issue a client code. Should be same as client_code in case of client itself doing transaction.
channel_code Y Enum Identifies the channel used by client to initiate request to the platform. Eg : MOBILE_APP, WEB_APP

Rsponse HTTP Header Parameters

Parameter (Y/N/C) Type Description
content-Type Y Alphanumeric MIME type of response message’s body
Content-Hash Y AlphaNumeric Salted Hash of http response body. hash=SHA-256(salt+JsonResponseBody)

Rsponse HTTP Body Parameters

Parameter (Y/N/C) Type Description
redirect_url C AlphaNumeric ESP redirect URL. You have to redirect user to this url to complete the esign transaction. Mandatory in case of esp_sdk_channel is WEB
esp_request_xml C AlphaNumeric Base64 encoded request xml that should be passed to ESP android sdk. Mandatory in case of esp_sdk_channel is ANDROID
